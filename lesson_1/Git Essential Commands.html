Git & Version Control

Essential Commands & Workflows

Technical Onboarding for Data Teams

February 2026

What is Git?

  • Distributed version control system
  • Created by Linus Torvalds in 2005
  • Tracks changes in source code
  • Enables seamless team collaboration

Why Use Git?

  • Complete project history and version tracking
  • Easy collaboration with team members
  • Branch and experiment safely
  • Industry standard for code management

Installing Git on macOS

Method 1: Download from official site

Visit git-scm.com and download the installer

Method 2: Install via Homebrew

brew install git

Installing Git on Windows

Download Git for Windows from git-scm.com

Run installer and follow wizard

Verify installation in Git Bash:

git --version

Initial Git Configuration

Set your name and email (required for commits):

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

These identify your commits in project history

Git Core Concepts

  • Repository: Project folder tracked by Git
  • Commit: Snapshot of your changes
  • Branch: Parallel version of code
  • Remote: Server copy (GitHub, GitLab)

Creating a Repository

Initialize new repository:

git init

Clone existing repository:

git clone <repository-url>

Basic Workflow: Status & Add

Check status of files:

git status

Stage files for commit:

git add <filename>    # or git add .

Basic Workflow: Commit

Commit staged changes:

git commit -m "Descriptive message"

Stage and commit together:

git commit -am "Message"

Remote Repository Commands

Add remote repository:

git remote add origin <url>

View remotes:

git remote -v

Sync with Remote: Push & Pull

Push commits to remote:

git push origin main

Pull latest changes:

git pull origin main

Creating & Switching Branches

Create new branch:

git branch feature-name

Switch to branch:

git checkout feature-name

Branch Management

Create and switch in one command:

git checkout -b feature-name

List all branches:

git branch

Merging Branches

Switch to target branch, then merge:

git checkout main
git merge feature-name

Combines changes from feature branch into main

Viewing Commit History

View commit log:

git log

Compact one-line view:

git log --oneline

Undoing Changes

Discard unstaged changes:

git restore <filename>

Unstage a file:

git restore --staged <filename>

Viewing Changes

See unstaged changes:

git diff

See staged changes:

git diff --staged

Best Practices

  • Commit often with clear messages
  • Pull before you push
  • Use branches for new features
  • Keep main branch stable

Team Collaboration Tips

  • Write descriptive commit messages
  • Review code via pull requests
  • Handle merge conflicts promptly
  • Document workflows in README

Typical Daily Workflow

  1. Pull latest changes: git pull
  2. Create feature branch
  3. Make changes and commit
  4. Push and create pull request

Command Quick Reference

  • git status - Check working tree status
  • git add . - Stage all changes
  • git commit -m "" - Commit with message
  • git push - Upload to remote
  • git pull - Download from remote
